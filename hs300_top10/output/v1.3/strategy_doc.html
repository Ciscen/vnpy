2.3 ATR 自适应止损机制

不同于固定止损百分比，ATR止损根据每只股票的真实波动幅度自动调整：

effective_stop = max(atr_stop_min, min(atr_stop_max, atr_multiplier * ATR14 / entry_price)) 示例: 低波动股（银行）: ATR14/price = 1.2% → 止损 = max(2%, 2.4%) = 2.4% 中波动股（消费）: ATR14/price = 2.5% → 止损 = max(2%, 5.0%) = 5.0% 高波动股（科创）: ATR14/price = 4.0% → 止损 = min(5%, 8.0%) = 5.0%（被上限截断）

这确保了：低波动股止损更紧（减少无效持仓），高波动股止损更松（避免被噪声震出）。

2.4 个股冷却期机制（V1.3 核心创新）

问题: V1.2 中，688303.SSE 被买入 6 次，累计亏损 -33.6%，仅 17% 胜率。模型反复选中正在下跌的股票。

解决方案: 当某只股票因止损卖出后，10 个交易日内不再买入该股票。

if "stop_loss" in sell_reason: stock_cooldowns[symbol] = 10 # 10个交易日冷却 # 每日递减 for symbol in stock_cooldowns: stock_cooldowns[symbol] -= 1 # 买入时过滤 buy_candidates = [s for s in candidates if s not in stock_cooldowns]

效果:

  • 阻止了 62 笔低质量买入
  • 释放了 43 笔替补买入（表现更优）
  • 仅减少 37 笔总交易（924→887），属于精准减负

五、注意事项

5.1 回测局限性

  • 资金规模: 10万资金下仓位集中度高，每只股票仅100股，个别大涨标的影响被放大。实盘需验证更大资金规模下的表现。
  • 滑点模型: 使用固定 0.2% 滑点，实际交易中小市值股票的冲击成本可能更高。
  • 涨停板: 已加入涨停板过滤（≥18%/9%不买），但实际涨停板封单量判断更复杂。
  • 国庆行情: 2024-09 单月 +31.6% 来自极端行情，不可预期重现。排除后年化约 25%。

5.2 实盘建议

  • 建议初始资金 ≥ 50万，确保每只股票至少可买 200-500 股
  • 周一收盘后运行模型，周二集合竞价下单
  • 密切关注止损触发后的冷却期列表，不要手动干预
  • 每月末检查模型信号质量（信号分布、正例率变化）
