## V1.3 个股冷却期优化（2026-05-06）

### 背景

基于 V1.2 的深度交易分析，识别到两个核心问题：

**问题 1: 首日/次日止损交易是最大亏损源**

| 持仓天数 | 交易数 | 总收益 | 胜率 | 分析 |
|----------|--------|--------|------|------|
| 1天 | 25笔 | **-170.2%** | **0%** | 全部亏损，首日即触发止损 |
| 2天 | 47笔 | **-156.6%** | 26% | 大部分亏损 |
| 3天+ | 361笔 | +1,124.5% | 57% | 盈利来源 |

**1-2天被止损的交易累计亏损 -327%，是策略最大拖累。**

**问题 2: 个别股票反复亏损**

| 股票 | 交易次数 | 累计亏损 | 胜率 |
|------|----------|----------|------|
| 688303.SSE（科创板） | 6笔 | -33.6% | 17% |
| 688047.SSE | 5笔 | -19.9% | 0% |
| 302132.SZSE | 3笔 | -18.6% | 0% |
| 688111.SSE | 5笔 | -18.5% | 20% |

### 废弃实验

**实验 A: 动量过滤 + 提高信号门槛 + 个股冷却（方案 C 组合）**

- 启用 momentum_filter（过滤近3天跌幅>-3%的股票）
- min_signal_prob 从 0.15 提升至 0.20
- stock_cooldown_days = 10

| 指标 | V1.2 | V1.3-expA | 变化 |
|------|------|-----------|------|
| 总收益率 | 93.30% | 87.31% | -5.99% |
| 最大回撤 | -17.72% | **-30.53%** | -12.81% |
| Sharpe | 1.30 | 1.09 | -0.21 |

**分析**: 全面劣化。核心原因：
1. 动量过滤与模型的均值回归特性冲突——XGBoost 选中的股票恰恰是近期下跌后预期反弹的标的
2. 提高信号门槛减少了分散化，5天满仓盈利交易从216笔降至167笔
3. 三个过滤器叠加过于激进，同时去掉了好交易和差交易

**结论**: 动量过滤和信号门槛提高在此策略中有害。**废弃。**

**实验 B: 绝对止损上限 absolute_stop_cap=0.08**

- 仅在 V1.2 基础上添加 8% 绝对止损

| 指标 | V1.2 | V1.3-expB | 变化 |
|------|------|-----------|------|
| 总收益率 | 93.30% | 93.30% | 0% |
| Sharpe | 1.30 | 1.30 | 0 |

**分析**: 完全无效。原因：ATR 自适应止损（2-5%）总是先于 8% 绝对止损触发。大亏损（如 -23%）是单日跳空造成的，前一天未达到任何止损阈值，当天跳空后两个止损同时触发，结果相同。

**结论**: ATR 止损已是最优止损机制，绝对止损上限无法额外保护。**废弃。**

### 最终方案: 个股止损冷却期

**核心思路**: 某只股票触发止损后，10 个交易日内不再买入该股票，避免反复在同一只下跌中的股票上亏损。

**实现**:
- 新增参数 `stock_cooldown_days: int = 10`
- 在 `on_trade` 中，当卖出原因包含 `stop_loss` 时，记录该股票的冷却期
- 每日递减冷却计数器
- 在买入候选过滤中排除冷却期内的股票

**仅此一项修改，无其他参数变化。**

### 参数变化（相对 V1.2）

| 参数 | V1.2 | V1.3 |
|------|------|------|
| stock_cooldown_days | 0（无） | **10** |

| 总收益率 | 93.30% | **119.49%** | +26.20% |
| 年化收益率 | 46.26% | **59.25%** | +12.99% |

## V2.0 月度选池 + 信号驱动持仓（2026-05-06）

## 标签泄漏修复 & 涨停板过滤

### 标签泄漏修复

**问题**: 滚动训练中，每月最后 `horizon` 天的标签隐式使用了预测月的未来价格数据。

**影响范围**:

## 统计审计（2026-05-08）

### 审计动机

V1.4 年化 96.7%、Sharpe 1.75 的结果"不太可信"，需要排查常见的量化回测陷阱。

### 审计清单（10 项）

| # | 检查项 | 结论 | 风险 |
|---|--------|------|:---:|
| 1 | **Look-Ahead Bias（前视偏差）** | Alpha158 特征全部回溯型，无泄漏 | 低 |
| 2 | **Survivorship Bias（幸存者偏差）** | V1.4 已用 BaoStock PIT 修正 | 低 |
| 3 | **Overfitting（过拟合）** | XGBoost 500 trees/depth=6 参数偏aggressive | 中 |
| 4 | **交易成本** | 双边 0.4%（0.2%佣金+0.2%滑点），合理偏低 | 中 |
| 5 | **Data Snooping（数据窥探）** | V1.0→V1.4 多次在同一回测期调参 | **高** |
| 6 | **Selection Bias（选择偏差）** | 仅展示最优版本，废弃版本未计入 | 中 |
| 7 | **仓位过度集中** | top_k=5 单票占 20%+，个别大涨贡献过大 | **高** |
| 8 | **回测引擎假设** | 涨跌停 ±10%（创业板/科创板应 ±20%）、停牌可交易 | **高** |
| 9 | **标签定义** | "周内 high 触及 +5%" 过于乐观，实际无法在最高点卖出 | **高** |
| 10 | **信号时序** | T 日收盘决策 → T+1 开盘成交，正确 | 低 |

### 修复内容（按优先级）

#### P0: 涨跌停模型修复

`vnpy/alpha/strategy/backtesting.py` 中 `cross_order` 函数：

| 板块 | 修复前 | 修复后 |
|------|--------|--------|
| 主板 | ±10% | ±10%（不变） |
| 创业板 (300/301) | ±10% | **±20%** |
| 科创板 (688) | ±10% | **±20%** |

```python
def _cn_a_share_limit_multipliers(vt_symbol: str) -> tuple[float, float]:
    symbol, _ = extract_vt_symbol(vt_symbol)
    if symbol.startswith(("300", "301", "688")):
        return 1.2, 0.8   # ±20%
    return 1.1, 0.9       # ±10%
```

#### P1: 停牌处理修复

虚拟 K 线（数据缺失时自动填充的 O=H=L=C, volume=0）不再允许撮合交易：

```python
def _is_forward_filled_no_trade_bar(bar: BarData) -> bool:
    if bar.volume != 0:
        return False
    # O=H=L=C 且 volume=0 → 视为停牌/缺失数据
    return abs(o - h) <= tol and abs(o - l) <= tol and abs(o - c) <= tol
```

#### P2: 标签定义改进（核心发现）

**原标签 (high_touch)**: 周内最高价 ≥ 周二开盘 × 1.05 → label=1

**问题**: 模型学会了预测"周内会冲高 5%"，但实际交易中你不可能在最高价卖出。持有到周五收盘时，冲高后的股票可能已经回落。

**新标签 (friday_close)**: 周五收盘 ≥ 周二开盘 × 1.03 → label=1

```python
# labeler.py: generate_weekly_labels_realistic()
last_close = closes[horizon_idx[-1]]  # 周五收盘价
label = 1 if last_close >= tuesday_open * 1.03 else 0
```

**标签对比影响**（V1.4 参数不变）:

| 标签 | 年化 | Sharpe | 最大回撤 | 含义 |
|------|:---:|:---:|:---:|------|
| high_touch (+5%) | 96.7% | 1.75 | -30.4% | 不可执行的乐观上界 |
| **friday_close (+3%)** | **29.0%** | **0.95** | **-18.9%** | 可执行的保守估计 |

> **这是审计中最关键的发现**: 标签从"冲高 5%"改为"持有到期收 3%"，年化从 97% 降到 29%。
> 说明原始模型学到的 alpha 中，约 2/3 来自无法实际捕获的日内冲高。

#### P3: 样本外验证框架

新增 `--oos-validate` 参数，支持拆分调参期/验证期:

```

验证期 (out-of-sample): 2025-05-01 ~ 2026-04-30
```

## V1.5 保守标签 + 参数优化（2026-05-08）

### 设计动机

审计发现 V1.4 的高收益（年化 97%）主要源于乐观标签。切换到保守标签 `friday_close` 后，V1.4 原参数仅年化 29%、Sharpe 0.95。

**核心问题**: 原参数（SL=4%, TP=4%, top_k=5）是为 `high_touch` 标签优化的，不适合 `friday_close` 标签。需要在新标签下重新调参。

### 调参方法

**方案选择**:
- 方案 A: 全参数网格搜索（768+组合，耗时过长）
- 方案 B: Optuna 贝叶斯优化（黑盒感强）
- **方案 C: 分阶段逐维扫描**（选用，可解释、速度快）

**执行**: `hs300_topk/tune_params.py`

信号已缓存（`friday_close` 训练不需重做），每次回测仅需 ~3.5 秒。分 5 个阶段逐维扫描，共约 100 次回测，总耗时 ~6.5 分钟。

| Stage | 扫描参数 | 搜索空间 | 最优值 |
|-------|----------|----------|--------|
| 1 | top_k | [3, 4, 5, 6, 8, 10] | **4** |
| 2 | stop_loss × tp_activate | 5×5=25 组合 | **SL=0.02, TP=0.03** |
| 3 | max_hold × tp_trail | 5×5=25 组合 | **hold=5, trail=0.015** |
| 4 | min_signal_prob × dynamic_k_thresh | 6×6=36 组合 | **prob=0.0, dk=0.30** |
| 5 | stock_cooldown × max_replace_ratio | 5×3=15 组合 | **cool=10, replace=0.5** |

**评分函数**: `score = sqrt(Sharpe × 收益回撤比)`，兼顾风险调整收益和回撤控制。

### 参数变化（V1.4 → V1.5）

| 参数 | V1.4 | V1.5 | 变化说明 |
|------|:---:|:---:|------|
| 标签 | high_touch (+5%) | **friday_close (+3%)** | 可执行收益 |
| top_k | 5 | **4** | 更集中 |
| stop_loss_pct | 0.04 | **0.02** | 更紧止损 |
| tp_activate_pct | 0.04 | **0.03** | 更早激活 |
| tp_trail_pct | 0.02 | **0.015** | 更紧追踪 |
| max_replace_ratio | 0.7 | **0.5** | 更平滑换仓 |
| min_signal_prob | 0.15 | **0.0** | 不过滤弱信号 |
| max_hold_days | 5 | 5 | 不变 |
| stock_cooldown_days | 10 | 10 | 不变 |

**参数变化的内在逻辑**: 保守标签下单笔预期收益更小，因此需要更紧的风控（SL 2%/TP 3%）来快速锁定利润，更平滑的换仓（50%）来减少摩擦。

### 回测结果

| 指标 | V1.4 旧标签 | V1.4 新标签 | **V1.5 新标签** |
|------|:---:|:---:|:---:|
| **年化收益** | 96.7% | 29.0% | **78.3%** |
| **Sharpe** | 1.75 | 0.95 | **1.68** |
| **最大回撤** | -30.4% | -18.9% | **-17.5%** |
| **收益回撤比** | 3.24 | 2.30 | **4.83** |
| 总交易笔数 | 583 | 548 | 442 |
| 回测区间 | 24.05-26.04 | 24.05-26.04 | 24.05-26.04 |

### OOS 验证（严格样本外）

**流程**: 仅在 in-sample 窗口（2024.05-2025.04）搜索最优 top_k 和 SL/TP → 用找到的参数在 OOS 窗口（2025.05-2026.04）验证。

| 指标 | IS 调参 (24.05-25.04) | **真OOS (25.05-26.04)** | V1.4原参 OOS |
|------|:---:|:---:|:---:|
| 年化收益 | 57.6% | **48.9%** | 36.6% |
| Sharpe | 1.49 | **1.70** | 1.34 |
| 最大回撤 | -17.5% | **-11.7%** | -11.1% |
| 收益回撤比 | 3.17 | **2.77** | 2.53 |

**关键发现**:
1. IS 最优参数与全样本调参结果完全一致（top_k=4, SL=0.02, TP=0.03） → 参数选择稳健
2. OOS Sharpe 1.70 **高于** IS 的 1.49 → 无参数过拟合迹象
3. OOS 年化 48.9% 高于 IS 的 57.6% 是因为 OOS 期间（2025.05-2026.04）市场波动更有利

### 经验教训

1. **标签定义是回测收益的最大杠杆**: high_touch → friday_close 让年化从 97% 降到 29%，单这一项变化就值 3 倍差异
2. **参数优化在正确标签上仍能大幅提升**: 同一个 friday_close 标签，V1.4 参数年化 29%，V1.5 优化后 78%（提升 2.7 倍）
3. **极端事件主导短期高 Sharpe**: 2 年内仅 6 笔 pnl>20% 的交易贡献了 71% 的盈亏，统计意义有限
4. **真 OOS 与 IS 高度一致**: 说明模型和参数没有严重过拟合，但 1 年 OOS 样本量仍不够充分
5. **中性预期年化 25-35%**: 这是一个更诚实的基准，可以作为实盘表现的合理预期

### 关于"不可信"的回答

V1.5 年化 78% 仍然很高，原因是:
1. **2024-2026 是结构性牛市**: 沪深300 指数同期上涨约 20%，策略的超额收益来自选股 alpha
2. **集中持仓放大了正负偏差**: 4 只股票意味着单票暴涨/暴跌都会大幅影响组合
3. **极端事件贡献过大**: 国庆行情 + 2025 夏季行情贡献了 63% 的收益

**真实评估**: 排除极端事件后年化约 **25-35%**，在 A 股量化策略中属于"优秀但非超级明星"的水平，是一个**可以作为起点进入实盘**的策略，但不应期望持续年化 50%+。
