本文档梳理 **V1.4（生产基线）** 与 **V1.4R（熊市防御向）** 及网格实验中的全部方案：调参动机、实现要点、回测设定与结果表格。数据来自仓库内脚本的一次或多次跑批（见「复现实验」）。

## 2. 调参思路与时间线（为何要 V1.4R）

### 2.1 问题定义

- **V1.4 + Lag-3** 在近年样本外（偏牛）环境表现突出，已成为生产基线。
- **单向做多 + 周度换股** 在系统性下跌中仍会出现大回撤；2022–2023 熊市区间上，基线回撤与亏损幅度显著。
- 目标：**在不破坏牛市弹性的前提下，尽量改善极端行情下的回撤与尾部风险**；若无法兼顾，则明确 trade-off，并用网格量化。

### 2.2 方案演进（概念层）

1. **方案 A：Regime 健康度分 → 仓位缩放**  
   用沪深300指数多维度打分（趋势 + 波动 + 动量），`regime_filter=True` 时：`effective_cash_ratio ≈ cash_ratio * sqrt(regime_score)`，且在 `regime_score < regime_min_score` 时周一清仓并停止加仓。
2. **方案 B：组合回撤熔断**  
   当组合净值相对历史峰值回撤超过 `max_portfolio_drawdown` 时，当日清仓；进入 `drawdown_cooldown_days` 日冷却；**冷却结束日将峰值重置为当前净值**（避免「熔断一次、永久不再交易」的实现 bug）。
3. **方案 C：指数均线门控**  
   `use_market_filter=True`：指数收盘低于 `market_ma_period` 日均线时，周一不再持有（与 Regime 独立，可叠加）。

调参顺序：**先横向网格 Regime / 熔断 / 阈值**，再在优胜底座上**叠加 MA**，最后把**默认预设**收敛为**仅用 R6 式熔断**（见第 5、6 节）。

## 3. 实现要点（与调参直接相关的代码行为）

实现位置：[`hs300_topk_strategy.py`](../strategy/hs300_topk_strategy.py)。

### 3.1 Regime 分数 `_compute_regime_score`

- 输入：基准指数收盘价序列（默认 `market_benchmark = 000300.SSE`），长度需覆盖 `regime_ma_long`、`regime_vol_baseline` 等窗口。
- **趋势**（约 1/3 权重）：现价相对短期均线、长期均线是否在上方。
- **波动**（约 1/3 权重）：短期已实现波动相对长期基线；**波动越高，分项越低**。
- **动量**（约 1/3 权重）：中期收益映射到 [0,1]。
- 输出：`regime_score ∈ [0,1]`；数据不足时返回 `1.0`（不拦交易）。

### 3.2 仓位缩放（仅 `regime_filter=True`）

- `scale = sqrt(regime_score)`（缓和非牛市环境下的过度砍仓，但仍显著压低牛市收益）。
- 应用于 `_full_rebalance` / `_smooth_rebalance` 中买入侧资金分配。

### 3.3 周一 Regime 硬清仓

- 若 `regime_filter=True` 且 `regime_score < regime_min_score`：已持仓中非当日风控卖出的标的标记为 `regime_exit` 并清仓；本周不再建新仓。

### 3.4 组合回撤熔断 `_portfolio_drawdown_check`

- `max_portfolio_drawdown > 0` 时：维护 `_portfolio_peak`；回撤达标 → 清仓 + 进入冷却。
- **冷却计数逐日递减**；**减至 0 当日**将 `_portfolio_peak` 设为当前估算净值，从新一轮峰值重新跟踪（修复早期「永远 True、长期空仓」问题）。

### 3.5 回测基准数据

- `use_market_filter=True` 或 `regime_filter=True` 时，主流程与网格脚本会将 `000300.SSE` 加入 `vt_symbols` 并 `add_contract_setting`（见 `run_pipeline.py`、`run_regime_grid.py`、`run_regime_grid2.py`）。
- **仅组合回撤熔断**（`regime_filter=False` 且 `use_market_filter=False`）时，策略不依赖指数 K 线；预设 **OPTIMIZED_V14R** 当前即如此。

## 7. 中间实验记录（便于排查历史误解）

### 7.1 早期「Regime + 熔断」版 V1.4R 与熔断实现 bug

- 曾出现：组合回撤熔断后 **peak 不随冷却结束重置**，导致 **长期空仓或重复触发**，全周期结果异常偏小。
- **修复**：冷却结束当日 `_portfolio_peak = 当前净值`（见策略源码）。

### 7.2 `run_regime_test.py`：修复后仍带 Regime 的 V1.4R 与纯 V1.4 对比（存档）

在将 **OPTIMIZED_V14R** 改为「仅 R6」之前，若仍开启 `regime_filter` + `sqrt` 缩放，多区间曾出现：

- 熊市：回撤可明显收窄，但 Sharpe 未必最优；
- 牛市：年化与总收益 **大幅低于** V1.4（典型从 ~73% 年化量级压到 ~49%），与网格结论一致。

具体数字以当时终端输出为准；**当前仓库预设已不采用该组合**。

### 7.3 `run_bearish_test.py`

- 用途：熊市区间 2022-01～2023-12、Lag-3 训练后，**快速对比 `OPTIMIZED_V14` 与 `OPTIMIZED_V14R`**（脚本内 `configs` 以当前文件为准）。
- 更系统的多方案扫描请使用 `run_regime_grid` / `run_regime_grid2`。

## 8. 最终选型与使用建议

| 场景 | 推荐预设 | 说明 |
|------|-----------|------|
| 生产主策略（与当前 Lag-3 研究一致） | `OPTIMIZED_V14` | 无组合熔断，牛市弹性满 |
| 希望在统计上改善「峰值回撤触发后的减仓节奏」、且接受略多牛市区间交易行为变化 | `OPTIMIZED_V14R`（R6） | **仅 15% 组合回撤熔断 + 10 日冷却** |
| 极限防守研究 | 自定义 `replace(OPTIMIZED_V14, use_market_filter=True, market_ma_period=60, …)` | 参考 grid2 **R6+MA60** 类，明确牺牲牛市 |

**重要声明**：上表为历史样本内 / 分区样本外回测结果，**不预示未来收益**；尤其 **2024–2026 牛市区间** 与 **2022–2023 熊市区间** 环境差异极大，任何单一区间「最优」都可能是环境运气。
